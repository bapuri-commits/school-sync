## 작업 내용

### P1 이슈 수정 (3건)
- **#1 timeout 누락**: `BrowserSession.start()`에서 `set_default_navigation_timeout` / `set_default_timeout`을 `REQUEST_TIMEOUT * 1000`ms로 설정. 모든 extractor의 `page.goto()`에 자동 적용됨 — 개별 파일 수정 불필요.
- **#2 날짜 형식 불일치**: `normalizer.py`에 `_normalize_date_str()` 헬퍼 추가. `YYYY.MM.DD`, `YYYY/MM/DD` 등을 `YYYY-MM-DD`로 통일. eClass·포탈·학과 공지, 출석 기록 전부 적용.
- **#3 calendar IndexError**: `_semester_start_timestamp()`에서 split 실패 방어 추가. `normalize_calendar()`에서 개별 이벤트를 try/except로 감싸고 `evt["time_start"]` 직접 접근 제거.

### P2 이슈 수정 (3건)
- **#7 timetable 해석 가이드**: `ask.py` system prompt에 "요일+교시" 형식 해석법 + 교시별 시간표(1~9교시) 추가.
- **#8 max_tokens**: 2048 → 4096 증가.
- **#9 TimetableEntry 정리**: `models.py` TimetableEntry를 실제 nDRIMS 데이터 구조(schedule, room 등)에 맞게 재설계. `NormalizedOutput`에 `timetable` 필드 추가. `_normalize_ndrims_timetable()`이 `TimetableEntry` 모델 객체를 반환하도록 변경. timetable이 `schedule/timetable.json`에 모델 기반으로 저장됨.

### 추가 수정
- **syllabus 정규화 통합**: `SyllabusEntry` 모델 추가, `_normalize_syllabus()` 함수 추가, `NormalizedOutput.syllabus` 포함, `academics/syllabus.json` 저장, `ask.py`에 카테고리·키워드(교재, 교수, 강의계획 등) 등록.

- **models.py 정리**: 미사용 `time` import 제거.

## 핵심 결정 & 맥락

### 프로젝트 방향성 확정
학교생활 전반을 도와주는 에이전트가 필요한 상황. The_Agent가 최종 목표지만 규모가 방대해서 완성까지 시간이 오래 걸림. 그래서 **school_sync를 중심으로 학교생활 어시스턴트를 먼저 만들고, 나중에 The_Agent에 통합**하는 방향으로 확정.

**이유:**
1. school_sync가 이미 데이터 계층(4사이트 크롤링 + 정규화 + LLM Q&A) 완비
2. lesson-assist가 이미 school_sync 데이터에 의존
3. The_Agent Phase 3 = school_sync 연동 — 지금 잘 만들면 통합 비용 최소화
4. 즉시 사용 가능한 실용적 가치 + 개발 경험 축적

**포지셔닝 전환**: school_sync는 "크롤러"가 아니라 "학교생활 데이터 허브 + 어시스턴트"로 확장.
