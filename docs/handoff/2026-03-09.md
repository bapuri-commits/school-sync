# school_sync Handoff — 2026-03-09

## 현재 상태
school_sync 크롤링/정규화/Q&A는 안정 동작 중. 이번 세션에서는 Gemini와 논의한 "University Life OS" 설계안을 검토하고, 파이프라인 개편 방향을 논의함.

## 작업 내용
- Gemini와 브레인스토밍한 설계 문서를 `docs/GEMINI.md`로 저장
- GEMINI.md를 school_sync 실제 현황(DESIGN.md, ask.py, lesson-assist)과 대조 분석
- 다글로 도입 / NotebookLM 도입이 기존 파이프라인에 미치는 영향 논의

## 핵심 결정 & 맥락

### 배경
Gemini는 프로젝트명(school_sync, lesson-assist)이나 현재 구현 상태를 모르는 상태에서 방향성만 논의한 것. GEMINI.md는 그 기록이지 설계 문서가 아님.

### 논의 결론: 두 가지 파이프라인 변경 검토 중

#### 1. 다글로로 faster-whisper 대체 (lesson-assist)
- **방향**: lesson-assist의 전사 엔진을 로컬 faster-whisper → 다글로(외부 서비스) export 기반으로 전환
- **장점**: 한국어 정확도 향상, GPU/CUDA/torch 의존성 제거, 품질교정(review.py) 단계 축소
- **트레이드오프**: 전사가 fully-auto에서 semi-auto로 변경 (다글로 export → 폴더에 넣기)
- **영향 범위**: transcribe.py, _transcribe_worker.py, review.py, preprocess.py 제거 → SRT/TXT 임포터 신규 개발
- **합의**: 대체할 만하다는 데 동의. 구체적 구현은 추가 논의 필요

#### 2. NotebookLM 학습 인프라 도입
- **방향**: "학사 관리"(ask.py)와 "학습 심화"(NotebookLM)를 역할 분리
  - ask.py → 성적/시간표/과제/졸업 등 행정 질의 (유지)
  - NotebookLM → 강의 내용 딥다이브, 시험 대비 (신규)
- **핵심**: API가 없으므로 아키텍처 레이어가 아니라 "출력 포맷 추가"로 접근
- **lesson-assist 산출물에 NotebookLM 최적화 export 추가** (Obsidian 노트와 병렬)
- **합의**: 방향은 동의하나 세부 사항 추가 논의 필요

### 예상 개발 범위 (lesson-assist 개편)
1. 다글로 SRT/TXT 임포터 (transcribe.py 대체)
2. PDF↔전사본 Context Matcher (school_sync 다운로드 자료 + 전사본 매핑)
3. NotebookLM 최적화 export (obsidian_writer.py와 병렬)
4. 기존 유지: 요약(GPT-4o), 액션 추출, 데일리 연동

### 예상 파이프라인
```
[수업 전] school_sync --download → PDF/PPT 다운로드
[수업 중] 다글로 녹음/전사
[수업 후] 다글로 export → 특정 폴더
            ↓
         lesson-assist (개편)
           ├── SRT 임포트 + 정제
           ├── PDF 텍스트 추출 + 페이지별 인덱싱
           ├── 전사본 ↔ PDF 매칭
           ├── GPT-4o 요약 + 액션 추출
           └── 출력:
                ├── Obsidian 강의 노트
                └── NotebookLM용 통합본 [과목_날짜].md
[학습] NotebookLM 업로드 → 딥다이브
[학사] ask.py → 행정 질의 (기존 유지)
```
