# school_sync Handoff — 2026-03-09

## 현재 상태
v1.0 안정. 학습 컨텍스트 생성 모듈(`context_export.py`) 추가 완료. lesson-assist와의 책임 경계 재정립됨.

## 작업 내용
- `context_export.py` 신규 생성 — 과목별 학습 컨텍스트 마크다운을 `output/context/{과목}.md`로 생성
  - 강의계획서(structured format), 과제/마감, 공지사항 포함
  - YAML frontmatter에 과목명, 생성시각, 현재 주차/토픽 메타데이터 포함
  - 학사일정에서 학기 시작일 추정하여 주차 매칭 시도
- `main.py` 수정 — 정규화 완료 후 `export_context()` 자동 호출 (`--normalize-only`에도 적용)
- `docs/DESIGN.md` 갱신 — 아키텍처도에 context_export 레이어 추가, lesson-assist 관계 설명 갱신

## 핵심 결정 & 맥락
- **프로젝트 역할 확정**: school_sync = 데이터 수집 + 가공 + 챗봇 (연구/튜닝 주력), lesson-assist = 패키징 + 포맷팅 + 전달 도구
- **context_builder 로직 이동**: lesson-assist에 있던 `context_builder.py`를 school_sync `context_export.py`로 이동. lesson-assist는 school_sync가 생성한 파일을 읽기만 함
- **데이터 가공 경계**: school_sync가 모든 데이터 가공(수업자료, 과제, 시험정보 통합)을 담당. 향후 PDF↔전사본 매칭, 토픽 분할 등 고급 가공도 여기서 수행
- **NotebookLM과 ask.py 역할 중복 허용**: ask.py=학사질문, NotebookLM=강의학습. 과제/시험 등 일부 중복은 허용
