# school_sync Handoff — 2026-03-13

## 현재 상태
StudyHub 웹 서비스 구현 + VPS 배포 완료. `study.syworkspace.cloud`에서 운영 중.

## 작업 내용
- school_sync + lesson-assist를 통합한 **StudyHub** 웹 서비스 구현 (5 Phase)
  - Phase 1: FastAPI 백엔드 (대시보드, 과목, 데이터 API)
  - Phase 2: React/Vite 프론트엔드 (대시보드, 과목 상세 6탭)
  - Phase 3: AI Q&A SSE 스트리밍 (ask.py 래핑, 채팅 UI)
  - Phase 4: 동기화 제어 (크롤링/정규화/패키징 트리거, 실시간 로그)
  - Phase 5: Docker 배포 (Playwright + Chromium, 멀티스테이지)
- SyOps JWT 쿠키 인증 + permissions.yaml 유저별 권한 시스템
- materials.py board 소스 캐시 스킵 버그 수정
- lesson-assist config.yaml v2 형식 업데이트
- VPS 배포: Docker + nginx + Cloudflare DNS + SyOps 서비스 등록

## 핵심 결정 & 맥락
- **독립 서브도메인 선택**: 기능 규모가 크고 두 프로젝트 통합이라 SyOps 내 경로 대신 `study.syworkspace.cloud` 독립 배포
- **SyOps 쿠키 공유 인증**: `.syworkspace.cloud` 도메인 쿠키로 별도 로그인 없이 인증
- **permissions.yaml 기반 권한**: admin 전체 접근, user별 개별 권한 설정 (DB 불필요)
- **서비스 통합**: 기존 lesson-assist(8202) + school-sync(8203) 분리 계획을 StudyHub(8203) 하나로 통합
